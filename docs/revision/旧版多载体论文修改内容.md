# 旧版多载体框架论文修改内容

本文修稿主线按旧版实现收敛为：符号、标点、空格三类载体的多载体弹幕隐蔽通信框架。后续实现中的 compact in-text carrier、humanized phrase carrier 和浏览器最小流量演示不写入本轮论文主线，以避免论文描述与旧版代码实现不一致。

## 一、论文定位修改

建议将论文定位从“完全隐蔽/不可检测通信”调整为：

```text
一种面向直播弹幕公开文本流的多载体隐蔽通信框架，重点解决弹幕短文本、平台清洗、部分丢失、顺序扰动和连续会话恢复问题。
```

也就是说，论文主张应强调：

- 多载体承载能力；
- 在符号、标点、空格局部损伤下的恢复能力；
- 冗余副本和投票带来的鲁棒性；
- 弹幕公开流中的端到端可恢复性；
- 隐蔽性是“降低显著固定模式”，不是“完全不可检测”。

不建议继续使用以下绝对表述：

```text
undetectable
completely covert
cannot be detected
perfect stealthiness
```

建议替换为：

```text
reduce obvious carrier artifacts
improve stream-level concealment
mitigate fixed-pattern exposure
provide a configurable robustness-concealment trade-off
```

## 二、摘要修改建议

可将摘要相关部分改为如下英文：

```text
Live streaming bullet comments form a public, real-time and highly interactive text stream, which provides a potential carrier for covert communication but also introduces practical challenges such as short comment length, platform-side filtering, partial message loss, order perturbation and session synchronization. To address these challenges, this paper proposes a multi-carrier covert communication framework for live streaming bullet comments. The framework maps protocol packets into three types of bullet-comment carriers, namely symbol-based, punctuation-based and space-based carriers, and uses redundant replicas and voting-based recovery to improve robustness under text-layer perturbations. The sender encodes the protected payload into unified protocol packets and schedules them across multiple carrier types, while the receiver extracts candidate packets from the public bullet-comment stream and reconstructs the payload through grouping and weighted voting. Experiments under simulated lossy bullet-comment channels show that the proposed multi-carrier design improves recovery reliability under carrier damage, message loss and order perturbation. We further add a detectability analysis and discuss the robustness-overhead-concealment trade-off, showing that the framework reduces fixed carrier-pattern exposure but should not be interpreted as perfectly undetectable.
```

对应中文解释：

```text
摘要里要突出三件事：第一，直播弹幕是公开实时短文本流；第二，本文方法是符号、标点、空格三类多载体；第三，实验重点是恢复鲁棒性和可检测性权衡，而不是声称完全不可检测。
```

## 三、贡献点修改建议

原贡献点可以改成四条：

```text
1. We design a unified multi-carrier packet mapping scheme for live streaming bullet comments. The scheme supports symbol-based, punctuation-based and space-based carriers under a shared packet structure.

2. We introduce redundant packet replicas and weighted voting recovery to improve robustness against text-layer perturbations, including carrier damage, comment loss and order perturbation.

3. We design a lightweight session advancement and periodic resynchronization mechanism to support continuous communication over unstable public bullet-comment streams.

4. We add a detectability and trade-off analysis. Instead of claiming perfect undetectability, we report rule-based and statistical detection results and discuss the trade-off among robustness, overhead and concealment.
```

中文讲法：

```text
贡献点不要再写成“隐蔽性完全安全”，而是写成“统一多载体编码、冗余投票恢复、连续会话支持、检测与权衡分析”。
```

## 四、方法部分修改内容

### 4.1 多载体映射机制

可加入或替换为以下段落：

```text
The proposed framework adopts three carrier types that naturally appear in live streaming bullet comments: symbol-based carriers, punctuation-based carriers and space-based carriers. These carriers differ in their visible representation, but they share the same protocol packet structure. Therefore, the sender can schedule packets across different carriers without changing the receiver-side recovery logic.

For symbol-based carriers, decimal digits are mapped to visible symbols. For punctuation-based carriers, decimal digits are mapped to common punctuation marks. For space-based carriers, decimal digits are represented by different whitespace patterns. Since live bullet comments may experience platform-side cleaning, the three carrier types provide complementary robustness. When one carrier type is damaged or partially filtered, replicas carried by other types may still remain recoverable.
```

中文解释：

```text
这里重点讲：符号、标点、空格只是外在表现不同，底层协议包结构一致，所以接收端可以统一识别和恢复。
```

### 4.2 统一协议包结构

可加入：

```text
To make different carriers recoverable under a unified rule, each payload segment is first converted into a fixed-length digit sequence and then organized as protocol packets. A packet contains a type field, a sequence field and a payload field. The type field indicates the packet category, the sequence field provides ordering information, and the payload field carries the encoded content. This structure allows the receiver to group candidate observations by packet type and sequence index before applying voting-based recovery.
```

如果保留旧版公式，可在公式后解释：

```text
The key point of this structure is that carrier selection and packet recovery are decoupled. The carrier only determines how the packet appears in the public bullet-comment stream, while the packet fields determine how it is reconstructed at the receiver.
```

中文解释：

```text
载体负责“看起来像什么”，协议包负责“接收端怎么恢复”。这两个层次要分开讲。
```

### 4.3 冗余副本与投票恢复

可加入：

```text
To tolerate message loss and local carrier damage, each protocol packet is transmitted with redundant replicas. These replicas may be assigned to one or more carrier types. At the receiver side, candidate packets with the same type and sequence index are grouped into the same recovery cluster. Since some replicas may be damaged or incorrectly parsed, the receiver applies weighted voting within each cluster. The candidate with the highest score is selected as the recovered packet for the corresponding structural position.
```

中文解释：

```text
这里回应审稿人关心的鲁棒性：不是靠单条弹幕成功，而是靠多个副本和投票恢复。
```

## 五、隐蔽性评估补充内容

由于本轮论文按旧版多载体走，隐蔽性部分要谨慎写。建议新增一个小节：

```text
Stealthiness and Detectability Analysis
```

可写：

```text
In addition to decoding reliability, we evaluate whether the generated bullet comments introduce visible or statistical artifacts. We extract comment-level features including comment length, carrier-symbol count, carrier-punctuation count, whitespace-run length and loaded-comment density. We further evaluate two types of detectors: a rule-based detector that targets fixed carrier patterns, and a statistical z-score detector trained from benign bullet-comment statistics.

The results show that fixed and repeated carrier patterns can be detected by simple rules, which indicates that robustness alone is insufficient for evaluating a covert bullet-comment channel. The multi-carrier scheduling strategy reduces dependence on a single visible carrier type and allows protocol packets to be distributed across symbol, punctuation and space carriers. However, loaded comments may still differ statistically from benign comments because they contain protocol-bearing information. Therefore, we report both loaded-only and stream-level results and present the method as a configurable robustness-overhead-concealment trade-off rather than a perfectly undetectable channel.
```

中文解释：

```text
这里一定要承认：旧版多载体不是完全不可检测。我们补充检测实验，是为了把结论改得更严谨。
```

## 七、局限性段落

建议在 Discussion 或 Conclusion 前加入：

```text
The current framework improves robustness by distributing protocol packets across multiple bullet-comment carriers and by using redundant voting-based recovery. However, this design does not imply perfect undetectability. Protocol-bearing comments may still introduce statistical differences in length, symbol usage, punctuation usage or whitespace patterns, especially under a strong adversary that can isolate loaded comments. In addition, increasing redundancy improves recovery reliability but also increases public comment overhead and loaded-comment density. Therefore, the proposed framework should be understood as a practical robustness-oriented covert communication framework with an explicit concealment-overhead trade-off. Future work will investigate adaptive carrier scheduling and online room-style learning to further reduce statistical artifacts in real live-room contexts.
```

中文解释：

```text
这段很重要。它主动承认旧版系统边界，能缓和审稿人对“不可检测性不足”的质疑。
```

## 八、审稿回复口径

### 对“隐蔽性不足”的回复

```text
We agree that the original manuscript overemphasized decoding robustness and did not sufficiently evaluate detectability. In the revision, we add a stealthiness and detectability analysis based on rule-based and statistical detectors. The revised manuscript reports both loaded-only and stream-level views. The loaded-only view represents a strong adversary that can isolate protocol-bearing comments, whereas the stream-level view evaluates the mixed public bullet-comment stream. The results show that the proposed multi-carrier scheduling can reduce dependence on a single fixed carrier type, but loaded comments may still remain statistically distinguishable. We therefore revise the claim and present the framework as a robustness-oriented multi-carrier covert communication method with an explicit robustness-overhead-concealment trade-off, rather than as a perfectly undetectable channel.
```

### 对“缺少比较实验”的回复

```text
We add protocol-level baseline comparisons under the same lossy bullet-comment channel model. The baselines include a timing-modulation channel and a natural-comment choice channel. These baselines are implemented as protocol-level reproductions rather than the original authors' code. The revised evaluation compares decoding success rate, average number of comments and detectability-related metrics under clean, reorder-only and lossy settings.
```

### 对“多载体贡献不清楚”的回复

```text
We clarify that the main role of the multi-carrier design is carrier complementarity under text-layer perturbations. Symbol, punctuation and space carriers share the same packet structure but are affected differently by platform cleaning and text normalization. By scheduling replicas across multiple carriers and applying weighted voting at the receiver, the framework improves recovery stability when one carrier type is partially damaged.
```

## 十、最终结论建议

论文最终结论可改为：

```text
This paper presents a multi-carrier covert communication framework for live streaming bullet comments. By mapping unified protocol packets into symbol-based, punctuation-based and space-based carriers, and by combining redundant replicas with voting-based recovery, the framework improves message recovery under public text-layer perturbations. The revised evaluation further shows that robustness, overhead and concealment are coupled: increasing redundancy improves recovery but increases public comment overhead, while carrier diversity reduces dependence on a single visible pattern but does not guarantee perfect undetectability. These results indicate that multi-carrier bullet-comment communication is feasible under controlled lossy-channel settings, while future work is still needed to improve adaptive concealment in real live-room environments.
```
