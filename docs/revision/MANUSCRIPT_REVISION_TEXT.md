## 1. Add To Evaluation Metrics

In addition to decoding robustness, we evaluate the statistical concealment of the generated bullet-comment stream. Let \(B\) denote a benign bullet-comment set and \(C\) denote a generated covert-comment set. For each comment, we extract text-level features including comment length, carrier-symbol count, carrier-punctuation count, whitespace count, maximum whitespace-run length and loaded-comment indicators. Distributional deviation is measured using Jensen-Shannon divergence (JS) on discrete feature distributions. We also evaluate two detectors: a rule-based detector that checks whether a comment ends with a fixed four-carrier-symbol pattern, and a statistical z-score detector trained from benign-comment feature statistics. Detector performance is reported using precision, recall and F1 score.

We report both loaded-only and stream-level views. The loaded-only view evaluates only comments that contain protocol-bearing fragments and represents a strong adversary that can already isolate candidate comments. The stream-level view evaluates the mixed public stream containing both loaded comments and ordinary distractor comments and represents an observer who only sees the overall bullet-comment stream. Reporting both views avoids overstating concealment and allows the robustness-concealment trade-off to be analyzed explicitly.

## 2. Add Stealthiness And Detectability Subsection

### Stealthiness and Detectability Analysis

To address whether the proposed carrier mapping introduces observable statistical artifacts, we conduct a detectability experiment using benign bullet-comment templates and covert streams generated by the implemented sender. The original fixed-symbol suffix design was first used as a diagnostic baseline. In that version, each loaded comment ended with a fixed four-symbol carrier pattern. This pattern was trivially detectable: the rule-based detector achieved precision 1.0000, recall 1.0000 and F1 1.0000. This confirms that robustness alone is insufficient and that carrier scheduling must be evaluated together with statistical concealment.

We therefore replace the fixed four-symbol suffix with a short-fragment, low-density scheduling strategy. Each four-digit protocol code is split into two short fragments. The sender uses punctuation-prioritized mixed carriers and inserts ordinary bullet comments between protocol-bearing comments. The receiver reconstructs the protocol code by extracting and reassembling the short fragments. This design removes the fixed four-symbol ending pattern: the rule-based detector obtains F1 0.0000 in both loaded-only and stream-level evaluations.

The stream-level distribution becomes closer to benign bullet comments after the scheduling adjustment. Under the balanced setting \(FRAGMENT\_REPLICAS=3\) and \(FILLERS\_PER\_PAYLOAD=2\), the stream-level z-score detector obtains F1 0.6667 and the length JS divergence is 0.1226 under the sequence-indexed redundant configuration. However, the loaded-only comments remain distinguishable because sequence-indexed redundant fragments carry metadata as well as payload digits. This result shows that the proposed method should not be interpreted as perfectly undetectable. Instead, concealment, robustness and overhead form a measurable trade-off.

## 3. Add Sequence-Indexed Redundancy Description

### Sequence-Indexed Redundant Fragment Recovery

To improve recovery under bullet-comment loss, we add sequence-indexed redundant fragments at the protocol layer. Each four-digit protocol code is divided into two two-digit fragments. A transmitted fragment record is formatted as

\[
R=(s_1,s_2,f,d_1,d_2),
\]

where \(s_1s_2\) is a two-digit sequence index, \(f\) is the fragment index within the protocol code and \(d_1d_2\) is the two-digit fragment payload. Each fragment record is mapped to the selected carrier symbols and repeated \(r\) times. At the receiver side, observed fragment records are grouped by sequence index and fragment index. Majority voting is applied within each group, and the two fragment payloads are concatenated in fragment-index order to reconstruct the original four-digit protocol code.

This mechanism prevents a single missing bullet comment from shifting the parsing boundary of all subsequent fragments. It also makes recovery less dependent on strict arrival order, because sequence indices provide structural alignment after public-stream perturbation. The additional metadata and repetitions increase the number of bullet comments, so the redundancy level must be selected according to the required robustness and concealment level.

## 4. Add External Baseline Comparison Subsection

### Comparison With Protocol-Level Baselines

We compare the proposed method with two protocol-level baseline reproductions under the same public text-layer channel. The first baseline is a time-modulation channel, where information is represented by short and long inter-comment intervals with repeated bits. The second baseline is a natural-comment choice channel, where predefined natural comments represent two-bit symbols with repeated observations. These baselines are not the original authors' implementations; rather, they reproduce the main carrier assumptions of time modulation and natural-language/comment-choice channels under the same loss and reordering model.

The comparison uses 50 trials per condition and the message `CovLBCG_test_123#`. Under clean and reorder-only conditions, all methods recover the message successfully. Under light loss (\(p_{loss}=0.02\), reorder jitter 0.10), the proposed method reaches 1.0000 success after sequence-indexed redundancy, whereas both simplified baselines fail because loss shifts their repeated timing or comment-choice groups. Under the default lossy condition (\(p_{loss}=0.10\), reorder jitter 0.10), the proposed method achieves 0.9200 success, while the two baselines remain at 0.0000 in this reproduction.

## 7. Add Limitations Paragraph

The current implementation improves stream-level concealment but does not make loaded comments indistinguishable from benign comments under a strong adversary that can isolate protocol-bearing comments. In particular, sequence-indexed redundancy introduces metadata-bearing fragments, which improve recovery under loss but increase the statistical difference of loaded comments. Therefore, the proposed framework should be understood as a configurable covert communication framework with an explicit robustness-overhead-concealment trade-off, rather than as a perfectly undetectable steganographic channel. Future work will investigate semantic carrier placement and adaptive template selection to reduce loaded-comment distinguishability while preserving loss recovery.

## 8. Add Room-Style Learning Stage

### Room-Adaptive Style Learning

Before transmitting in a target live room, the sender first observes recent public bullet comments from that room and builds a room-specific benign style profile. The profile includes cleaned comment templates, comment-length statistics, punctuation usage and frequent local expressions. The generated covert comments then use the learned room-specific templates as wrappers and filler comments instead of relying on a fixed global template library.

This design makes the template set adaptive to the current room. When the target room changes, the learning stage is repeated and a new room-specific template file and profile are generated. In the real-room corpus experiment, this adaptive strategy reduces stream-level length JS divergence from 0.3834 to 0.1054 and punctuation JS divergence from 0.4718 to 0.1914. The improvement is stream-level: loaded-only comments remain distinguishable because protocol metadata is still appended to carrier-bearing comments.

## 9. Add Real-Platform Feasibility Validation

### Real-Platform Feasibility Validation

To verify that the implemented sender and receiver can operate on an actual live bullet-comment stream, we also conduct a small-scale feasibility validation on Bilibili live room 6963590. This experiment is not used as the main statistical evidence for robustness or concealment, because public live rooms are not appropriate for high-volume repeated transmission. Instead, it verifies the end-to-end implementation path from generated bullet comments to platform delivery, live-stream observation, fragment extraction, reassembly and decryption.

In this validation, the sender transmits a short test message, `hi#`, using the room-adaptive wrappers learned from recently observed room comments. To minimize public traffic, the validation uses one fragment replica and no filler comments. The complete transmission consists of 17 bullet comments: one ordinary warm-up comment, one calibration marker, 14 protocol-bearing fragment comments and one termination marker. All send requests are accepted by the platform API. The receiver observes the comments from the actual live bullet-comment stream, extracts 14 mixed-carrier fragment records, reconstructs seven four-digit protocol codes with no missing sequence index and recovers the plaintext message `hi`.

This validation shows that the proposed implementation can complete a real-platform end-to-end round trip. The controlled offline experiments remain the basis for the reported loss robustness, baseline comparison and detectability analysis.

## 11. Add Compact In-Text Carrier Mapping

### Compact In-Text Carrier Mapping

To further reduce the loaded-comment artifacts introduced by explicit fragment metadata, we replace the legacy five-character mixed-carrier suffix with a compact in-text mapping. Each sequence-indexed fragment record still represents the same logical fields, namely sequence index, fragment index and two payload digits. Instead of directly writing the five decimal digits as five carrier characters at the end of the comment, the sender packs the record into a base-12 representation over common punctuation characters and embeds the resulting four carrier characters into the wrapper text. The receiver supports both the compact in-text format and the legacy trailing format for backward compatibility.

This change keeps the redundancy and sequence-indexed reconstruction mechanism unchanged, so loss recovery is not affected. In our controlled benchmark, the proposed method keeps the same default-loss success rate of 0.9200 and the same average comment count of 361.00. The compact mapping reduces loaded-comment length mean from 12.2260 to 10.9940, removes carrier-symbol usage from loaded comments, and reduces stream-level length JS divergence from 0.1244 to 0.0727 in the compact-vs-legacy experiment. The loaded-only statistical detector remains strong, with F1 decreasing only from 0.9970 to 0.9881, so this is an incremental concealment improvement rather than a complete solution.

The room-adaptive setting benefits more because learned wrappers are shorter and less repetitive. With synthetic room-style wrappers, compact in-text mapping reduces loaded-only F1 from 0.9940 to 0.9615 while keeping stream-level F1 at 0.6667. This result motivates further work on semantic carrier placement, where punctuation choices are constrained by natural clause boundaries rather than inserted according to simple positional rules.

We also validate the compact mapping on the authorized real Bilibili browser/WebSocket path. Using the learned popular-room template file in room 23087172, the sender transmits `hi#` as 17 browser-input comments with one fragment replica and no filler comments. The receiver observes the calibration marker, extracts 14 compact-carrier fragments, reconstructs seven protocol codes with no missing sequence index and recovers `hi`. This validation is reported only as a low-flow implementation check; the statistical detectability conclusions remain based on the controlled repeatable experiments.
